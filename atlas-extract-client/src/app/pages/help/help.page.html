What is ATLAS Extract and how to use it

Getting an account

Access to ATLAS Extract is by invitation only. You cannot sign up on your own — please contact the administrator to have an account created for you. Once your account is ready you will receive your login credentials and can sign in from the login page.

What does the app do?

ATLAS Extract is a tool for turning guide books (PDFs) into structured, machine-readable data. You upload a guide book, describe the information you want to pull out of it, and the app uses AI to read every page and extract individual entries — things like sights, places of interest, or any other items described in the book — each with a title, description, category, and tags.

The result is a clean JSON file you can download and use however you like — import it into a database, feed it into another tool, or process it further.

Step by step

1. Upload a source

Go to Upload Source and upload a PDF guide book. Give it a friendly name so you can identify it later. Only PDF files are supported.

2. Extract data

Go to Extract Data, find your source and click Extract. On the extraction page you can:

  • Preview the PDF directly in the browser.
  • Extract a sample of the text so you can see how the AI will read it.
  • Describe the structure — tell the system what language the book is in, what topic it covers, how long descriptions should be, and what fields each entry should have. The better your description, the better the output.
  • Run a sample extraction on a small page range to check the results before committing to the full document.
  • Start the full extraction. The app splits the book into page ranges, sends each one to the AI, and saves the results. You can watch each batch complete in real time.

3. Review extractions

Go to Extractions to see all your past and in-progress extraction runs. Click Details on any completed extraction to browse the extracted entries in a table and download the full JSON.

4. Enrich the data (optional)

Go to Enrich Data to add extra information to your extracted entries:

  • GPS coordinates — the app looks up each entry by name and country and attaches geographic coordinates (latitude, longitude) along with country, state and county.
  • AI-generated images — the app generates a photorealistic image for each entry using its title and description. Both a full-size (1024 px) and a thumbnail (350 px) version are saved.

Enrichment runs in the background. You can watch the status of each entry update in real time. Once complete, download the enriched data as JSON — it includes all the original fields plus GPS and image storage paths.

A few things to keep in mind

  • Extraction and enrichment happen asynchronously in the cloud — closing the browser tab is fine, the job keeps running.
  • A running enrichment cannot be re-triggered. Once it finishes you can run it again if needed.
  • Image generation requires an OpenAI key configured by the administrator. If images are not available, the checkbox will still appear but results may fail.
  • GPS coordinates come from a geocoding service and are best-effort — accuracy depends on how well-known the location is.
